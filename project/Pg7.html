CSS Rules



How CSS rules are weighted:

In CSS, the most specific rule is applied to every element.

Analogy: US Legal System v. 4 yr old kid.

Federal (most broad) -> State -> Municipal -> Your Family (most specific)
(least weight).............->.........->................-> (most weight)

The house rules are going to be the ones that the kid lives by regularly, with very little impact felt from any Federal rules. The kid is more worried about crossing Mum & Dad than they are about crossing the President. (Terrible analogy, sue me.) Basically, National rules would have the weight of House rules, and vice versa, if the US was controlled like CSS.

This hierarchy is what is meant with the word 'Cascading'.

You go down as far as you need to do into the rules until you find the cone that best describes the elements, then that rule is applied. There is no need to memorise all possible attributes and values. Instead, one should get comfortable looking up CSS documentation. As my mate is fond of saying: "You don't need to know everything. You just need to know how to look everything up." It also matters where you define the rules and the order in which they are applied.

Vocabulary

Style:

Can refer to multiple things-
  • The HTML element, e.g.: <style>div {color:blue}</style>
  • The HTML attribute, e.g.: <div style="color : blue">text</div>
  • The general look of a site, e.g.: "I like the style of that site, it's fun but easy on the eyes."
  • A verb, e.g.: "The structure is done, but I jsut need to style it."

    • Placeholder:

    Can be thought of like 'find and replace' in Word. You've got a word that's holding a spot that soon another word(s) is/are going to occupy in its place. Like "resident" in a mail merge letter that eventually gets filled in with the name of the resident/recipient for that specific letter. E.g.: Placeholder "Body-part" -> Will later be replaced by Eye, or Hair

    Rule:

    A line of CSS code that describes a value that a certain attribute should have. E.g.: Rules: Do not brush wet, do not dye, dandruff shampoo only

    Property:

    The characteristic that you want to change.

    Value:

    The quality or quantity you want to assign to the attribute. E.g.: Values: blonde, dry, processed, virgin

    Class:

    Refers to a group of elements that can be styled together.

More Vocabulary:

ID:

IDs are unique identifiers that identify a HTML element.

Comments:

Comments are lines of text that the computer ignores. They are used to leave notes to yourself or to others who might check out your code in the future. -In HTML, code comments are prefixed with <!-- and end with --> Everything in between is ignored by the computer. -In CSS, code comments begin with /* and end with */. Again, everything in between those two markers is ignored.
